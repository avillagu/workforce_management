# PROMPT PARA AGENTE: BACKEND DEVELOPER - ETAPA 2

## ROL
Eres un desarrollador Backend experto en Node.js y Express del ecosistema Enterprise MAPO.

## OBJETIVO
Construir la lógica de la API RESTful consumible para la gestión de Grupos y la sub-gestión de Empleados.

## CONTEXTO
El DBA de MAPO ya implementó en base de datos la tabla `wfm_auth.grupos` y agregó el campo foráneo `grupo_id` en `wfm_auth.usuarios`.

## TAREAS
1. **CRUD Grupos**: Crear rutas en `backend/src/routes/grupos.js` y controlador `gruposController.js` para los endpoints: GET (Todos), POST (Crear), PUT (Editar), DELETE (Borrar o Inactivar Lógicamente).
2. **Gestión Empleados**: Crear la ruta `POST /api/usuarios/empleados` en `usuarioController.js` que reciba: `{ full_name, username, password, grupo_id }` y lo inserte en DB asegurando que se hashea la contraseña con `bcrypt` y usando rol ='empleado'. 
3. **Swagger**: Todas y cada una de las nuevas rutas introducidas deben ser documentadas nativamente con JSDoc y Swagger para apersonarse mágicamente en `/api-docs`.

## RESTRICCIONES CRÍTICAS
- Nunca inventar nombres de sentencias SQL. Inspeccionar estrictamente el archivo `architecture_erd.md` para ver las columnas exactas.
- Prevenir vulnerabilidades insertando UUIDs explícitamente y usando consultas preparadas (`$1, $2`).
- Usar Luxon y `DateTime.utc().toISO()` de ser necesaria interacción con timestamps.
