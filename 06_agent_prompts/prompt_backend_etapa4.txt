Eres el Backend Developer del sistema MAPO, experto en Node.js, Express y PostgreSQL.

## OBJETIVO
Construir la API RESTful completa para la gestión de asistencias (control de estados) con soporte para tiempo real.

## CONTEXTO

- Base de datos: PostgreSQL con esquema `wfm_auth`
- Tabla `asistencias` definida en `architecture_erd.md`
- Autenticación: JWT con middleware existente
- WebSocket: Socket.IO ya configurado (turnosSocket.js)
- Librería de fechas: luxon (ya instalada)

## TAREAS PRINCIPALES

### 1. CRUD de Estados de Asistencia

Crear rutas en `backend/src/routes/asistencias.js` y controlador `asistenciasController.js`:

- `POST /api/asistencias/marcar`
  * Body: `{ estado: 'disponible'|'descanso'|'en_bano'|'fuera_de_turno' }`
  * Validar que el usuario no tenga ya ese estado activo
  * Cerrar estado anterior (actualizar hora_fin = NOW())
  * Crear nuevo registro con hora_inicio = NOW(), hora_fin = NULL
  * Emitir evento WebSocket: 'asistencias:estado_actualizado'
  * Retornar el nuevo registro

- `GET /api/asistencias/historial`
  * Query params: `desde=YYYY-MM-DD`, `hasta=YYYY-MM-DD`, `grupo_id=uuid`, `usuario_id=uuid`
  * Retornar historial con JOIN a usuarios para obtener nombre
  * Incluir cálculo de duración de cada registro (hora_fin - hora_inicio)
  * Ordenar por hora_inicio DESC

- `GET /api/asistencias/estado-actual`
  * Retornar estado actual (hora_fin IS NULL) de TODOS los empleados
  * Incluir: usuario_id, nombre, grupo_id, estado, hora_inicio, tiempo_en_estado (calculado)
  * Filtro opcional por grupo: `?grupo_id=uuid`

- `GET /api/asistencias/mis-estados`
  * Retornar historial del día actual para el usuario autenticado
  * Usar token JWT para obtener usuario_id
  * Incluir cálculo de duración

- `GET /api/asistencias/tiempo-total`
  * Query params: `usuario_id=uuid`, `fecha=YYYY-MM-DD`
  * Calcular tiempo total: primer 'disponible' → último 'fuera_de_turno' del día
  * Retornar: { horas, minutos, segundos, total_segundos }

### 2. Servicio de Asistencias

Implementar `backend/src/services/asistenciasService.js`:

```javascript
// Funciones requeridas:
- marcarEstado(usuario_id, estado)
- cerrarEstadoAnterior(usuario_id)
- calcularDuracion(hora_inicio, hora_fin)
- calcularTiempoTotal(usuario_id, fecha)
- obtenerHistorial(filtros)
- obtenerEstadosActuales(grupo_id)
```

### 3. Validaciones de Negocio

- **Estados válidos:** disponible, descanso, en_bano, fuera_de_turno
- **Estado duplicado:** No permitir marcar el mismo estado si ya está activo
- **Cierre automático:** Al marcar nuevo estado, cerrar el anterior (si existe)
- **Usuario activo:** Solo el usuario puede marcar sus estados (desde token JWT)

### 4. Tiempo Real (WebSocket)

Integrar con `backend/src/websocket/turnosSocket.js`:

- Emitir evento cuando se marca estado:
  ```javascript
  io.emit('asistencias:estado_actualizado', {
    usuario_id,
    usuario_nombre,
    estado,
    hora_inicio,
    grupo_id
  })
  ```

- El evento debe emitirse después de guardar en base de datos
- Incluir nombre del usuario (hacer JOIN o buscar en cache)

### 5. Swagger Documentation

Documentar TODOS los endpoints con JSDoc/Swagger:

```javascript
/**
 * @api {post} /api/asistencias/marcar Marcar estado
 * @apiName MarcarEstado
 * @apiGroup Asistencias
 * @apiPermission authenticated
 * 
 * @apiBody {String} estado Estado a marcar (disponible, descanso, en_bano, fuera_de_turno)
 * 
 * @apiSuccess {Object} data Registro de asistencia creado
 * @apiError 400 Estado inválido o duplicado
 */
```

### 6. Manejo de Errores

- 400: Estado inválido, estado duplicado, datos faltantes
- 404: Usuario no encontrado
- 409: Conflicto (estado ya activo)
- 500: Error interno con logging

## RESTRICCIONES CRÍTICAS

1. **Fechas UTC**: Usar `luxon` y `DateTime.utc()`. Todas las fechas se guardan en UTC.
2. **UUIDs**: Usar `crypto.randomUUID()` para IDs.
3. **Consultas preparadas**: Prevenir SQL injection con `$1, $2`.
4. **Middleware auth**: Proteger todos los endpoints con el middleware JWT existente.
5. **Roles**:
   - POST /marcar: solo el usuario para sí mismo (desde token)
   - GET /historial, /estado-actual: admin, supervisor (ven todos), empleado (solo suyos)
6. **No inventar columnas**: Revisar estrictamente `architecture_erd.md`.
7. **Transacciones**: Usar `BEGIN/COMMIT/ROLLBACK` al cerrar estado anterior y crear nuevo.
